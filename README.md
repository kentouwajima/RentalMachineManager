## システム概要
本システムは、機材のレンタル状況や営業所情報を管理し、営業所間でのリアルタイムな情報共有を実現することを目的としています。

## 開発背景

### 従来の課題
- 各営業所が機材の貸し出し状況を個別に管理しており、他営業所の在庫状況を即時に確認できない。
- 電話やメールでの確認が必要で、対応に時間がかかるため機会損失が発生する。

### 本システムの導入による改善
- **リアルタイムな在庫確認**  
  本システムを導入し、営業所間でリアルタイムに機材のレンタル状況を確認可能に。

- **貸し出し管理の効率化**  
  レンタル状況を可視化し、スムーズな機材管理を実現。

- **データの整合性を確保**  
  機材情報と営業所情報を一元管理し、正確なデータ運用が可能に。
   

## 主な機能

| 機能 | 詳細 |
|------|------------------------------------------------|
| 機材の検索・一覧表示 | 機材名、メーカー、型式、ステータス、営業所でフィルタリング可能 |
| 機材の詳細表示 | 機材の詳細情報（メーカー・型式・現在のレンタル状況、営業所情報）を確認可能 |
| 機材の登録・編集・削除 | 機材情報（機材名・メーカー・型式・ステータス・営業所）の登録・更新・削除 |
| 営業所の登録・管理 | 営業所名と住所を記録・編集 |
| 機材のステータス管理 | 「レンタル中」「利用可能」「メンテナンス中」などの状態表示 |

# 今後の展望

## 1. 機能面での追加実装
今後、より利便性の高いシステムを目指し、以下の機能を追加予定です。  

### 予約機能の追加
- **現在の課題:** 機材の貸し出し状況はリアルタイムで確認できるが、事前予約ができないため、貸出予定を営業所間で共有しづらい。  
- **実装方法:**  
  - 予約テーブル（`reservations`）を追加し、予約データを管理  
  - 機材ごとに予約カレンダーを表示（Thymeleaf + FullCalendarを使用）  
  - 予約済み機材の貸し出しを制限するバリデーションを追加  

### レンタル情報の追加（顧客情報 + レンタル期間の管理）
- **現在の課題:** 現在のシステムでは機材の貸し出し状況を管理しているが、「誰に貸し出したか」「レンタル期間」などの情報が不足している。  
- **実装方法:**  
  - **顧客管理テーブル（`customers`）の追加**
    - 顧客名、連絡先、企業名などを記録  
  - **レンタル情報テーブル（`rentals`）の追加**
    - 機材ID、顧客ID、レンタル開始日・終了日、返却状況などを管理  
  - **UIの改善**
    - 機材詳細ページに「レンタル履歴」を表示し、過去の貸し出し情報を確認できるようにする  
    - 返却日が近づいた機材をダッシュボードで通知  

## 2. フロントエンド技術の採用
現在のフロントエンドは **Thymeleaf + Bootstrap** を使用していますが、**よりインタラクティブなUIを実現するために、React + TypeScript への移行** を予定しています。

### React + TypeScriptへの移行
- **現在の課題:** Thymeleafはシンプルで使いやすいが、ページ遷移が多く、リアルタイムなデータ更新が難しい。  
- **実装方法:**  
  - フロントエンドをReact + TypeScriptに置き換え、APIとの非同期通信（Axios）を活用  
  - Spring Bootとの連携はREST API（Spring Web）で実装  
  - 状態管理にはRecoilまたはZustandを使用し、機材データの管理を最適化  

## 3. AWSの追加サービスの使用
現在は **EC2, RDS, ALB, IAM** を使用していますが、**さらなる運用の安定化や利便性向上のために以下のAWSサービスを追加予定** です。

### Route 53（独自ドメインの取得）
- **現在の課題:** 現在はIPアドレスや無料のサブドメインを使用しているため、URLの管理が煩雑。  
- **実装方法:**  
  - Route 53を利用して独自ドメインを取得  
  - ドメインにALBを関連付け、負荷分散とSSL対応を強化  

### AWS Certificate Manager（SSL証明書の管理）
- **現在の課題:** HTTPS化を手動で設定する必要があり、証明書の更新管理が煩雑。  
- **実装方法:**  
  - AWS Certificate Manager（ACM）で無料のSSL証明書を取得・管理  
  - ALBと統合し、HTTPS通信を標準化  

### CI/CDパイプライン（CodePipeline + GitHub Actions）
- **現在の課題:** 現在は手動でデプロイを行っているため、本番環境への反映に時間がかかる。  
- **実装方法:**  
  - AWS CodePipelineを使用し、GitHubリポジトリと連携  
  - GitHub Actionsで自動ビルド・テストを実施し、デプロイの品質を向上  
  - AWS CodeDeployを活用し、EC2に対して自動デプロイを実装  

## まとめ
| 項目 | 追加予定の技術・機能 |
|------|----------------------|
| 機能面での追加実装 | 予約機能・レンタル情報（顧客情報 + レンタル期間管理） |
| フロントエンド技術の採用 | React + TypeScriptへの移行 |
| AWSの追加サービス | Route 53（独自ドメイン）・ACM（SSL証明書）・CI/CD（CodePipeline + GitHub Actions） |

最終目標として、シンプルな在庫管理システムから、より高度な「営業所間の機材管理プラットフォーム」へと進化させることを目指しています。
